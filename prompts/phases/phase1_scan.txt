
PHASE 1 — SCAN MARCHÉ
⏱️ HEARTBEAT START PHASE 1
_hb_start(1)

Univers de candidats de départ (toujours inclus) : portfolio_coins de config.json (BTC, STX, XRP, SOL, SUI) plus les coins listés dans universe_extra de config.json s'il existe. Limiter à universe_scan_top_n (20) coins au total. Exclure les stablecoins (USDT, BUSD, FDUSD, USDC, etc.).

Appelle les trois screeners TradingView pour identifier les opportunités sur le marché BINANCE :
1. mcp__tradingview__top_gainers(exchange="BINANCE", timeframe="4h", limit=25)
   → liste des coins avec le meilleur momentum sur 4h (change%, volume). Stocke la liste dans top_gainers_symbols.
2. mcp__tradingview__volume_breakout_scanner(exchange="BINANCE", timeframe="4h", volume_multiplier=2.0, price_change_min=3.0, limit=25)
   → liste des breakouts volume (symbol, change%, volume). Stocke la liste dans breakout_symbols.
3. mcp__tradingview__market_sentiment(symbol="BTC", category="crypto", limit=20)
   → sentiment global du marché crypto. Stocke le résultat dans market_sentiment_data.

Si un appel MCP échoue → continuer avec les données disponibles (top_gainers_symbols ou breakout_symbols = [] si l'appel a échoué), ne pas bloquer le cycle.

Vérification de tradabilité USDC — pour chaque coin candidat de l'univers de départ, exécute :
kraken ticker {COIN}USDC -o json
- Si `data["{COIN}USDC"]` présent → tradable. Prix = `data["{COIN}USDC"]["c"][0]`.
- Volume 24h USDC = `float(data["{COIN}USDC"]["v"][1]) * float(prix)`. Si volume < 5M USDC → ✗
- Si la commande échoue ou la paire est absente → ✗ (non tradable)
Note : BTC doit utiliser la paire XBTUSDC (pas BTCUSDC) — mapping à appliquer avant l'appel.

Filtre définitif : éliminer immédiatement les coins marqués ✗ (paire USDC introuvable ou volume insuffisant). Ces coins ne doivent PAS être inclus dans l'univers final transmis à la Phase 2. Seuls les coins avec ✓ passent à la suite.

Calcul du sentiment global :
- Utilise market_sentiment_data (issu de mcp__tradingview__market_sentiment) si disponible.
- Sinon, calcule depuis les données kraken ticker : compte les coins avec priceChangePercent > 0 (bullish_count) et < 0 (bearish_count)
- Si bullish_count >= 2 × bearish_count → sentiment = "Bullish"
- Si bearish_count >= 2 × bullish_count → sentiment = "Bearish"
- Sinon → sentiment = "Neutral"
- Stocke sentiment comme variable pour Phase 3 et Phase 7.

Stocke pour chaque coin tradable : prix_actuel, volume_24h, change_pct_24h (depuis screeners / kraken ticker). Ces données seront réutilisées en Phase 3 pour le scoring.

→ tg("📡 Phase 1 — Scan marché\nSentiment : {sentiment}\n{N} candidats identifiés : {liste}")
⏱️ HEARTBEAT END PHASE 1 — appel hb(1) immédiatement ci-dessous
hb(1, summary=f"Sentiment {sentiment}, {N} candidats")
