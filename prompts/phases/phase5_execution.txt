
PHASE 5 — EXÉCUTION AUTOMATIQUE
⏱️ HEARTBEAT START PHASE 5
_hb_start(5)

Construis /tmp/cycle___CYCLE_ID___phase5_input.json avec les ordres préparés en Phase 4 :

{
  "ordres_prepares": [
    {"coin": "BTC", "quantite": 0.001, "prix_entry": 60000.0, "prix_stop": 57600.0,
     "prix_tp": 64800.0, "montant_ordre": 60.0, "risk_usdc": 1.5,
     "stop_distance_pct": 0.04, "score": 8}
  ],
  "config": null
}

RÈGLES :
- Utilise UNIQUEMENT la liste ordres_prepares produite par la Phase 4 (ne pas inventer ou modifier ses valeurs)
- config: null pour laisser le script charger config.json lui-même

Puis exécute le script d'exécution :

```bash
__PROJECT_DIR__/.venv/bin/python3 __PROJECT_DIR__/binance-bot/core/phases/phase5_execution.py __CYCLE_ID__
```

Le script traite chaque ordre (dans l'ordre de score décroissant) : re-fetch prix et check drift
(skip TYPE_C), re-fetch solde USDC (skip TYPE_C), BUY MARKET + query du fill (3 tentatives, 1s
d'intervalle, skip TYPE_C si non rempli), clôture immédiate au marché si le prix dépasse déjà le
TP au fill (close_reason="market_above_tp_at_fill"), sinon pose du stop-loss (protection_failed +
alerte Telegram si échec). Il enregistre chaque trade dans state/trade_history.json et notifie
via tg() lui-même — pas de notification supplémentaire à envoyer côté orchestrateur.

Le script lit /tmp/cycle___CYCLE_ID___phase5_input.json et écrit /tmp/cycle___CYCLE_ID___phase5_output.json.
Stdout attendu : PHASE5_DONE|executed=X|skipped=Y

Lis /tmp/cycle___CYCLE_ID___phase5_output.json pour récupérer :
- executed, skipped, orders_executed, orders_skipped_detail

Mets à jour les variables de synthèse pour la suite du cycle :
executed = <valeur du output JSON>
skipped = <valeur du output JSON>
orders_skipped_detail = <valeur du output JSON>

⏱️ HEARTBEAT END PHASE 5 — appel hb(5) immédiatement ci-dessous
hb(5, summary=f"{executed} ordres exécutés, {skipped} skippés")
