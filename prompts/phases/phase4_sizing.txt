
PHASE 4 — SIZING ET PRÉPARATION DES ORDRES
⏱️ HEARTBEAT START PHASE 4
_hb_start(4)

Construis /tmp/cycle___CYCLE_ID___phase4_input.json avec les buy_candidates depuis Phase 3 :

{
  "buy_candidates": [{coin, score, signal_4h, prix_actuel, atr_pct}],
  "portfolio_total": <float — montant total du portefeuille depuis Phase 0>,
  "budget_disponible": <float — solde USDC disponible>,
  "config": null
}

Puis exécute le script de sizing :

```bash
__PROJECT_DIR__/.venv/bin/python3 __PROJECT_DIR__/binance-bot/core/phases/phase4_sizing.py __CYCLE_ID__
```

Le script lit /tmp/cycle___CYCLE_ID___phase4_input.json et écrit /tmp/cycle___CYCLE_ID___phase4_output.json.
Stdout attendu : PHASE4_DONE|orders=X|skipped=Y

Lis /tmp/cycle___CYCLE_ID___phase4_output.json pour récupérer :
- ordres_prepares : liste des ordres dimensionnés [{coin, quantite, prix_entry, prix_stop, prix_tp, montant_ordre, risk_usdc, score}]
- skipped : liste des coins ignorés [{coin, reason}]

Mets à jour les variables de synthèse :
- ordres_prepares, skipped (pour Phase 5)

⏱️ HEARTBEAT END PHASE 4 — appel hb(4) immédiatement ci-dessous
hb(4, summary=f"{len(ordres_prepares)} ordres dimensionnés, {len(skipped)} skippés")
